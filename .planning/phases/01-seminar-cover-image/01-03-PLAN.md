Create `frontend/src/features/seminars/components/CoverImageDropZone.tsx` as a self-contained component that handles eager upload. The component manages its own upload state and calls the mutations directly.

Props interface:
```typescript
interface CoverImageDropZoneProps {
  seminarId: string;
  currentImageUrl?: string;  // The relative URL from seminar.cover_image_url (e.g. "/uploads/seminars/abc.jpg")
}
```

The component does NOT take onImageChange callbacks — it calls `queryClient.invalidateQueries` via the mutation hooks, which causes useSeminarContext to re-fetch and update the displayed seminar automatically.

Implementation requirements:

**Empty state (no currentImageUrl):**
- Dashed border container with `border-2 border-dashed rounded-lg` styling
- Centered camera icon (lucide-react `Camera`, w-10 h-10, text-muted-foreground)
- Text below icon: "Arrastra una imagen aquí o haz clic para seleccionar" (text-sm text-muted-foreground)
- Subtext: "JPEG, PNG o WebP — máx. 5MB" (text-xs text-muted-foreground/70)
- Whole area is clickable (triggers hidden file input)
- Drag-over visual feedback: change border to `border-primary` and background to `bg-primary/5`

**Preview state (currentImageUrl set):**
- Show `<img>` filling the same container (aspect-video, object-cover)
- Construct full image URL: prefix the relative URL with the API base URL (check how apiClient is configured in `frontend/src/core/data/apiClient.ts` and use the same base URL constant)
- Overlay an X button (absolute positioned, top-2 right-2, rounded-full, bg-destructive text-white, w-8 h-8) — always visible, not just on hover
- Clicking X calls `useDeleteCoverImageMutation().mutateAsync(seminarId)` with a loading state on the button

**Upload loading state:**
- While uploading: show a centered `Loader2` spinning icon overlaid on the drop zone area (or the preview if replacing)
- Disable drag-and-drop and click during upload

**Client-side validation (runs before upload, no server round-trip):**
```typescript
const ALLOWED_TYPES = ['image/jpeg', 'image/png', 'image/webp'];
const MAX_SIZE = 5 * 1024 * 1024;

const validateFile = (file: File): string | null => {
  if (!ALLOWED_TYPES.includes(file.type)) {
    return 'Formato no permitido. Usa JPEG, PNG o WebP.';
  }
  if (file.size > MAX_SIZE) {
    return 'El archivo supera el límite de 5MB.';
  }
  return null;
};
```

**Error display:** Inline paragraph below the drop zone container — `<p className="text-sm text-destructive mt-1">{uploadError}</p>`. Cleared when a new file is selected.

**File input:** Hidden `<input type="file" accept="image/jpeg,image/png,image/webp" ref={inputRef}>` — triggered by clicking the drop zone.

**handleFile flow:**
1. Validate client-side → set uploadError if invalid, return early
2. Clear uploadError
3. Set isUploading = true
4. Call `uploadMutation.mutateAsync({ seminarId, file })`
5. On success: mutation's onSuccess invalidates 'seminars' query — component re-renders with new currentImageUrl from the parent
6. On catch: set uploadError to "Error al subir la imagen. Inténtalo de nuevo."
7. Finally: set isUploading = false

**Drag handlers:**
```typescript
onDragOver={(e) => { e.preventDefault(); setIsDragging(true); }}
onDragLeave={() => setIsDragging(false)}
onDrop={(e) => { e.preventDefault(); setIsDragging(false); const file = e.dataTransfer.files[0]; if (file) handleFile(file); }}
onClick={() => !isUploading && inputRef.current?.click()}
```
